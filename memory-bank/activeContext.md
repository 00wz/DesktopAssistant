## Корректировки после ревью

### 1. Unified Kernel Factory
- Убраны отдельные коннекторы для Azure OpenAI и других провайдеров
- Все провайдеры работают через единый OpenAI-совместимый коннектор с custom BaseUrl
- Создан IKernelFactory и KernelFactory по образцу пользователя

### 2. Offline TTS (Piper)
- Убрана зависимость от Azure Cognitive Services Speech
- Добавлена поддержка Piper TTS через ONNX Runtime
- TTS будет работать полностью офлайн

### 3. MCP через Semantic Kernel
- Исследована встроенная поддержка MCP в Semantic Kernel
- MCP tools конвертируются в SK plugins
- Убран кастомный IMcpService (будет использоваться SK интеграция)

### 4. Summary Node для суммаризации
- Добавлен MessageNodeType.Summary
- При сборке контекста алгоритм останавливается на Summary Node
- Метод BuildContextAsync в ConversationService реализует эту логику

## Принятые решения

### 1. Архитектура

- **Clean Architecture** с разделением на 4 слоя (Domain, Application, Infrastructure, UI)
- **MVVM** для UI слоя с использованием CommunityToolkit.Mvvm
- **Repository Pattern** для доступа к данным
- **Event-driven** подход для коммуникации компонентов

### 2. Технологический стек

| Компонент | Выбранная технология | Обоснование |
|-----------|---------------------|-------------|
| UI | Avalonia UI 11.3 | Кроссплатформенность, XAML-based, поддержка TrayIcon |
| AI | Semantic Kernel 1.32 | Официальный SDK от Microsoft, поддержка множества провайдеров |
| STT | Vosk 0.3.38 | Офлайн работа, стриминг, поддержка русского |
| TTS | Azure Speech + OpenAI TTS | Качественные голоса, стриминг |
| DB | SQLite + EF Core 9 | Легковесность, кроссплатформенность |
| Audio | NAudio 2.2 | Полная поддержка аудио на всех платформах |

### 3. Структура графа истории

Решено использовать **древовидную структуру** для истории диалогов:
- Каждое сообщение — узел графа
- Узел может иметь множество дочерних узлов (ветвление)
- "Ветка" — путь от корня до текущего активного узла
- Возможность переключения между ветками
